# 2026-05-18 回退 360p — 妥协高画质

## 用户决策

> "no 不行,我们还是恢复到一开始那个 360p 的那个方法吧,妥协高画质了。"

调研 + 多轮自测后,所有"登录+1080p / PoToken / ANDROID_VR / MWEB" 路径在 Mac IP 上全部 LOGIN_REQUIRED。Dart 生态全无成功 cookie 登录案例。WEB+player.js+BotGuard 路径要 8h+ 重写且后续每月维护。用户拍板放弃高画质,回到 yt_explode 360p 兜底。

## 改动

撤掉本日所有 Innertube/cookie/双 client 接入:

1. **[yt_video.dart::fetchMuxedStreams](../../lib/services/youtube/yt_video.dart)** — 只调 `_fetchMuxedOnce`,2 次 retry,失败 throw `yt_explode: <err>`
2. **`_streamClients`** 恢复 `[android, androidSdkless, ios, tv]`,不再用 androidVr/safari/mweb
3. **`_fetchMuxedOnce` `authEnabled`** 写死 `false`,禁用 split 高画质流
4. **[yt_video_page.dart::_startPlayback](../../lib/pages/yt_video/yt_video_page.dart)** — `setMediaHeader` 不再注入 Cookie

## 保留(代码留在仓库,下轮可复活)
