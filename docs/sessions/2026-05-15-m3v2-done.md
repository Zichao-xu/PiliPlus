## 设计要点

**不复用任何 B 站耦合组件**:
- ❌ 不用 `VideoDetailController`(`late` 字段强耦合 bvid/aid)
- ❌ 不用 `PlPlayerController`(单例 ok 但 view widget hard-bind VideoDetailController)
- ❌ 不用 `PlPlayer view` widget
- ✅ 直接用 `media_kit.Player.create()` + `media_kit_video.VideoController.create()` + `Video` widget

**单文件 page** (~270 行):
- `lib/pages/yt_video/yt_video_page.dart` — StatefulWidget 自管 Player 生命周期(initState 启 / dispose 释放),内联简介区
- 简介区不复用 `introduction/yt/view.dart`(后者 hard-bind VideoDetailController,只能等清理或留作 dead code)

**兜底设计**:
- 简介 fetch 和 player fetch 各自独立 try/catch
- 失败转 `_playerError` 字符串显示,**不挂 widget tree**
- 任一失败不影响另一边

## API 教训

`media_kit` 项目用的是 fork 版,API 形式:
- `Player.create(configuration: PlayerConfiguration(...))` 异步(不是 `Player()` 同步)
- `VideoController.create(player, configuration: ...)` 异步(不是 `VideoController(player)` 同步)
- 上游 README 示例的同步形式不适用

UA 设置(`player.setMediaHeader(userAgent: ...)`)用 iOS Safari UA,保险起见。M3v2 真机播放无问题,UA 是否起作用未单独验证。
