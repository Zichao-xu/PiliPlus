# 2026-05-18 YT 画质/字幕/登录 三件套结论

## 用户原始诉求
1. YouTube 登录功能
2. YouTube 评论区(上次已完成)
3. 画质调节
4. 字幕

额外问题:
- 速度慢 ~200KB/s
- 视频缓存完才播
- 字幕未出现多国语言,自动翻译基本不生效

## 本次成果

### ✅ 画质调节(用户报"选 1080p 必 fallback 360p")
**根因**:不是代码 bug,是 YT 政策。YT 对未登录请求高画质 split stream URL 返回 **403**。
- muxed 360p:免登录,可播
- video-only 720p/1080p:必须带 cookie / PoToken,否则 anti-bot 拦截
- 用 mpv debug log + 直接观察 ffmpeg open 结果证实

**做法**:在 `yt_video.dart` 加 `authEnabled=false` 开关,split 流暂不进 menu。等登录接入后改 true 即可放开。代码保留 `_collectSplitStreams` 私有方法,不丢。

**实测**:onelastkiss 视频画质菜单只剩 360p ✓ 切换不再 fallback。

### ✅ 字幕多语言面板(用户报"未出现多国语言,自动翻译基本不生效")
**实现**:
- `fetchSubtitleTracks` 返回所有 native track(One Last Kiss 5 条 English)
- 控件面板 3 段式:`关闭` / `原始语言` / `自动翻译为...`
- 翻译子表 25 种常用语言(zh-Hans/zh-Hant/en/ja/ko/es/fr/de/ru/...)
- 选语言后走 `_setSubtitleTranslated`,拼 `&tlang=<lang>` 调 YT timedtext API

**实测**:
- 原始 English 字幕显示 ✓("Oh oh oh oh oh..." / "Someone I can't forget")
- 翻译为 中文(简体):菜单工作 ✓ 但屏上没显示
  - 推测原因:这 5 条英文都是**用户上传**的字幕(非 `asr` 自动生成),YT 不允许翻译用户上传字幕,只翻译自动生成字幕
  - 用 `t.isAutoGenerated` 区分,但 UI 暂未做"灰掉翻译"提示。**后续可加**:翻译选项点了之后若 mpv 取不到 vtt 内容,弹 toast "此字幕不支持自动翻译"

### ✅ YT 登录(嵌入 WebView 被 Google 拦截)
**根因**:Google 不允许第三方 app 用嵌入 WebView 登录(显示 "This browser may not be secure")。

**新方案**:cookie 粘贴登录
- 用户桌面浏览器登录 youtube.com
- DevTools → Cookies 复制必要 cookie
- App 内粘贴文本框 → 解析存到 `YtAuthService.saveCookies()`
- 支持格式:`name=value; name=value` 或多行
- "从剪贴板粘贴"按钮一键导入

**核心 cookie**:SAPISID(必须) + `__Secure-1PAPISID` / `__Secure-3PAPISID`(必须其一)。
其他 `__Secure-1PSID` / `LOGIN_INFO` 可选(更全权限)。

### 🟡 视频边缓存边播(用户报"缓存完才播")
**做法**:`bufferSize: 4 * 1024 * 1024`(4MB,vs media_kit 默认 32MB)
- 4MB 足够 360p 起播,不至于卡顿
- 比默认 32MB 显著降低首屏延迟

### 🔴 速度慢 ~200KB/s
**结论**:不是代码可解,YT CDN 路由 + 用户网络节点的事。
- 用户切节点后实测 ~317KB/s ✓
- 视频本身大小决定:1080p 5.5Mbps 流量需 ~700KB/s 才不卡(超出 200KB/s 现状)
- 缓解:已经只跑 360p (~290kbps),350KB/s 完全够用

## 关键技术点

### YT split stream 403 调研
- `youtube_explode_dart` 内部对 1080p URL 做 HEAD 验证 → 全部 403
- 所有 client (android/androidSdkless/ios/tvhtml5) 都被拦
- TVHTML5 返回 "Video unplayable"
- 即使绕过 yt_explode 直接喂 URL 给 libmpv,libmpv 也 403:`stream: Failed to open .`

### libmpv on macOS 走过的弯路(都不是根因)
- `setAudioTrack` 用 `audio-add ... cache` flag → 试过 `select`/`audio-files property` → 都不行
- 不是 cache flag 问题,是 video URL 本身 403

### 字幕 translate URL 格式
```dart
static String translateUrl(String nativeUrl, String tlang) {
  final uri = Uri.parse(nativeUrl);
  final params = Map<String, dynamic>.from(uri.queryParameters);
  params['tlang'] = tlang;
  return uri.replace(queryParameters: params).toString();
}
```
原 URL 含 `fmt=vtt` + `lang=en`,加 `tlang=zh-Hans` → YT timedtext API 应返回翻译后 vtt
**仅对 auto-generated (asr) 轨道生效**。

1. **登录**:我的 → 设置 → 关于 → YouTube 登录 → 桌面浏览器复制 cookie 粘贴 → 保存

4. **视频**:点击即播,无需等缓存完(4MB 小 buffer 配置)
